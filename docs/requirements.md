# 要件定義書 - 賞味期限管理アプリ（仮称）

> ステータス: ドラフト v1.0
> 最終更新: 2026-03-26

---

## 1. プロダクト概要

### 1.1 コンセプト

**「年間6万円の食品ロスを、ゼロに近づける」**

カメラをかざすだけで賞味期限を自動読み取り。期限が近い食品を通知で知らせ、使い切れば「節約」として可視化する。入力の手間を徹底的に排除した、食品ロス削減 × 節約アプリ。

### 1.2 ターゲットユーザー

幅広く（ペルソナを絞らない）。以下のような人が主なユーザー層：

- 自炊するが忙しくて食材を腐らせがちな社会人
- 食費を抑えたい一人暮らし〜ファミリー層
- 食品ロス・SDGsに関心がある層
- 防災備蓄品の期限を管理したい人

### 1.3 市場背景

| 指標 | 数値 |
|------|------|
| 日本の家庭系食品ロス | 年間233万トン |
| 一世帯あたりの損失 | 年間約6万円 |
| アプリ市場規模（2025年） | 約14.7億ドル（CAGR 13-26%） |
| 政府の追い風 | 食品ロス削減推進法、SDGs 12.3 |

### 1.4 既存アプリの課題（＝参入機会）

| 課題 | 本アプリの解決策 |
|------|----------------|
| 賞味期限が手入力で面倒 | OCRでカメラから自動読み取り |
| バーコードDBの網羅性が低い | ローカルDBに蓄積。ユーザーが育てる |
| サブスク/広告がウザい | 完全無料 |
| 通知の先がない（使い切りへの導線） | 節約額の見える化でモチベーション維持 |

```
│        │                   │        │

│  期限:    2026/04/15                │  ← OCRから自動 or 手動入力

│                                     │

│  🏠 ホーム │  📷 スキャン │  📊 統計    │
└───────────┴───────────┴─────────────┘
```

#### スキャンのフロー

```
カメラ起動
  ├─ バーコード検出 → ローカルDBから商品名を検索
  │   ├─ ヒット → 商品名を自動入力
  │   └─ ミス → 手動入力（入力後DBに保存 → 次回から自動）
  │
  ├─ 日付テキスト検出（OCR）→ 賞味期限を自動入力
  │   └─ 複数の日付検出 → ユーザーが選択
  │
  └─ 両方検出したら下部に結果表示
      └─ 保管場所を選んで「登録する」
```

#### OCR 対応フォーマット

日本の食品ラベルで使われる主要な日付表記：

| フォーマット | 例 |
|------------|-----|
| YYYY.MM.DD | 2026.04.15 |
| YYYY/MM/DD | 2026/04/15 |
| YYYY年MM月DD日 | 2026年4月15日 |
| YY.MM.DD | 26.04.15 |
| YYYYMMDD | 20260415 |
| 年月のみ | 2026.04（日は月末として扱う） |

## 4. データモデル

### 4.1 FoodItem（食品）

| フィールド | 型 | 説明 |
|----------|-----|------|
| id | UUID | 一意識別子 |
| name | String | 商品名 |
| expiryDate | Date | 賞味期限 |
| storageLocation | Enum | 冷蔵庫 / 冷凍庫 / その他 |
| quantity | Int | 個数（デフォルト 1） |
| barcode | String? | バーコード（任意） |
| status | Enum | active / consumed / discarded |
| price | Int? | 価格（円、任意。節約計算用） |
| createdAt | Date | 登録日時 |
| consumedAt | Date? | 消費/廃棄日時 |

### 4.2 BarcodeProduct（バーコード辞書）

| フィールド | 型 | 説明 |
|----------|-----|------|
| barcode | String | バーコード（PK） |
| name | String | 商品名 |
| defaultExpiryDays | Int? | デフォルトの賞味期限日数（学習用） |
| category | String? | カテゴリ（乳製品、肉類 etc.） |

### 4.3 ConsumptionLog（消費ログ）

| フィールド | 型 | 説明 |
|----------|-----|------|
| id | UUID | 一意識別子 |
| foodItemId | UUID | 対象食品のID |
| action | Enum | consumed / discarded |
| price | Int? | 金額（節約 or ロス） |
| date | Date | 記録日時 |

## 5. 節約計算ロジック

### 5.1 基本ルール

```
通知が来た食品を期限内に「使い切った」= 節約

節約額 = 食品の価格（price）
ロス額 = 捨てた食品の価格（price）
使い切り率 = consumed / (consumed + discarded) × 100
```

### 5.2 価格が未入力の場合

食品カテゴリごとのデフォルト平均価格を使用：

| カテゴリ | デフォルト価格 |
|---------|-------------|
| 乳製品 | ¥200 |
| パン | ¥150 |
| 肉類 | ¥400 |
| 野菜 | ¥150 |
| 調味料 | ¥300 |
| その他 | ¥200 |

### 5.3 表示

- **今月の節約額**（ホーム画面サマリー + 統計画面）
- **累計節約額**（統計画面）
- **週別/月別の推移**（統計画面のグラフ）

## 6. 非機能要件

### 6.1 パフォーマンス

- OCR認識: 1秒以内にリアルタイム検出
- アプリ起動: 2秒以内
- スキャン→登録: 3タップ以内で完了

### 6.2 プライバシー

- カメラ映像はオンデバイス処理のみ（外部送信なし）
- 全データはiCloud経由でユーザーのアカウントに保存
- 自社サーバー不要（ランニングコスト ¥0）

### 6.3 対応デバイス

- iPhone: iOS 17.0+
- iPad: 対応予定（v2以降）
- Apple Watch: 非対応

### 6.4 ローカライゼーション

- v1: 日本語 + 英語
- 端末の言語設定に自動追従
- String Catalogs（.xcstrings）で管理

### 6.5 アクセシビリティ

- VoiceOver 対応
- Dynamic Type 対応
- カラーだけに依存しない情報設計（残り日数テキスト併記）
